Opis ideje, ki jo aplikacija podpira

Spletna stran je namenjena samo nadzoru in pomoči pri izgubi telesne teže. Ciljna publika so ljudje, ki želijo izgubiti težo ali pa le nadzorovati dnevno aktivnost. Aplikacija v primeri previsoke teže uporabniku poišče in na zemljevidu pikaže najbližje/najbolje ocenjeno aktivnost (fitnesi), ki ga lahko uporabimo tudi samis klikom na gumb "Pomagaj mi". Poleg tega nudi uporabniku tudi grafičen prikaz razmerja med njihovo aktivnostjo in indeksom telesne teže.

Uporabljeni zunanji viri podatkov

Uprabljen(i) API je (so):

Googlov API za zemljevide (uporabljena kombinacija API za html in javascript API) Do API se dostopa preko skripte na Googlov API strežnik. Ta se uporablja za to,da v okolici uporabnika najdemo najbolje ocenjen in najbližji fitness.

https://developers.google.com/maps/web/?hl=sl

Googlov API za risanje grafov. Skripta je na strežniku googla. API se uporablja za to, da izrišem 3 različne grafe, ki prikažejo spreminjane teže, drugi koliko so aktivni uporabniki in tretji odvisnost med indeksom telesne mase, aktivnostjo in količino spite vode.

https://developers.google.com/chart/

Master/detail načrtovalski vzorec

Master parametri se nahajajo pri grafih. Ko uporabnik izbere težo, aktivnost ali pa ITM z aktivnostjo in spito vodo spletna stran najprej posodobi množico, tako da gleda le za zadnjih 7 vnosov. Pri primeru ko izbere uporabnik težo ali aktivnost odfiltrira še vse ostale podatke ki jih ne potrebuje za izris grafa.

Uporabljena tehnika za vizualizacijo podatkov

Grafična prestavitev je bila narejena s pomočjo Google developer Charts API, ker sem ga uporabljal že v preteklosti in je uporaba relativno preprosta. Grafi ravno tako omogočijo dobro preglednost inizbiranje posamičnih točk za bolj natančno odčitovanje. Za predstavitev je bil izbran "tekoči" stolpični diagram, sam lahko na njem predstaviš več podatkov naenkrat brez da bi se vrednosti prekrivale.
